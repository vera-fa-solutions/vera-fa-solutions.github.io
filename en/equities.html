                                            Equities

                                            Equity is a form of instrument that indicates that the holder has proportionate ownership in the issuing company. Equities are usually bought and sold on stock exchanges. 

                                            Equities can be created, edited, and managed in FA Back. FA user interface provides you with relevant fields and relevant calculations for this type of instrument.

                                            Properties of Equities

                                            When you create a new Equity, there are predefined settings in FA that guide you through the process

                                            Code is something that you can freely define. It is a unique identifier of the security in the system.  

                                            Name - how you want to call thie stock (replace later all stocks by equities)

                                            Base type - for stock put in base type stock Decimals - it should be zero for stocks

                                            Group by type, Include in total and Future contract - ignore these. 

                                            Links to transaction types - a stock usually involves quite a lot of transaction types in it. There's a long list are the most common, Buy and Sell, and Divident are usually the most commonly used transaction types. -

                                            Subtype -

                                            Once these are set up, then managing a security becomes available in FAFor stocks different kinds of codes are usually quite important.

                                            ISIN Code - that allows you to identify the security in the market

                                            Trade code. it could be for example a Bloomberg ticker or the identifier of the security in an other place where this security is traded. Or To identify the security in the connector itself.These three codes can be used to import transactions or  prices to your securitiesupdate code are mainly used to connect to a market data connector

                                            The get update code         button allows you to fetch the update code with the ISIN code of the security. The update codes are mainly used to connect to a market data connector. Then the data you filled into Update code 2, eg, goes to a filed named Close 2 (in Market data info tab).

                                            Source is related to update code field. Usually, it is set to default, which means that when the system goes to the market data info, it goes from right to left and picks the leftmost one. In this case, it is some manually added data. 

                                            holiday calendar -  identifies between business days and nonbusiness days within the Securities market. 

                                            settlement date handling - how many days the security takes to settle. The result can be seen in Transaction window. 

                                            total outstanding amount - how many outstanding shares there are. supports calculating what we call a holding percent. So it tells you how much of the outstanding shares do you currently own. If the total outstanding amount is not defined then the holding present is shown as empty.

                                            marketplace is identified with a MIC code (Market identifier code) stock exchanges, where These Securities are traded in.

                                            settlement place is identified with a BIC code (bank identifier code),  the banks or financial institutions to Securities are settled in.

                                            Classifications - Fields class 1 - 5. First  you go to preferences and Define What kind of classification structures  I want to use. Then you can come into the security window and select the classification on your security. this information is very powerful in reporting. the more I classify, the more different viewpoints I get out of the reporting and analytics as well.

                                            Related corporate actions

                                            Related corporate actions for the security - trading with stocks is usually simple: you can buy, sell, hold it for a period, and get profit or sell it with loss.The most common corporate action related to stocks is a dividend - the company pay out profits to shareholders. in FA all the dividends should be managed with corporate actions. So we have a corporate action called dividend allows you to handle these kinds of situations. corporate actions is the place to go to manage the dividends In addition stocks are quite very often involved in different kinds of other corporate actions such as splits exchanges mergers Acquisitions share issues subscriptions and things like that.  when you're managing stocks as a security type there are you're quite heavily involved with corporate actions as well for different situations.
