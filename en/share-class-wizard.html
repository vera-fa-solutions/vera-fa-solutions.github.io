                        Share class wizard

                        The Share class wizard lets you create, edit or delete a fund share class. When you create a share class in the Fund management app, FA creates a share class security with the specified properties in FA Back.

                        To add a share class, click the button in the Fund details pane.

                        To edit a share class, click the button in the Share class details pane.

                        To delete a share class click Delete in the Share class wizard. You can delete a share class only if it doesn't have any transactions and isn't used in benchmarks.

                        Basic info

                        Share class details and NAV calculation settings.

                        Code*

                        A unique code defined for the share class. The code is used to identify the share class security and search for securities in the system. We recommend using numbers and letters (a-z) and avoiding special characters in the security code.

                        ISIN

                        The ISIN code of the share class security.

                        Name*

                        Name of the share class security.

                        Type*

                        Share class security type. Choose the type that is configured for funds in Preferences. If you have standard solution configuration, use Collective investment vehicles type.

                        Holiday calendar*

                        A holiday calendar that determines business days and holidays in the market to set a settlement date for subscriptions and redemptions. Holiday calendar options are configured in Preferences. You can also set the default holiday calendar used for new securities in Preferences.

                        Share class priority

                        The order in which share class NAVs are calculated if there are multiple share classes. The NAV for the share class with the highest priority is calculated first.

                        Decimal count for transaction units

                        Number of decimals used in the units field of the client transactions. If the field is empty, the number of decimals set for the security type is used.

                        Pricing

                        Unit price settings:

                        Currency

                        Share class currency – the currency share class units are quoted in.

                        Initial price

                        (Set for a new share class). Initial price of the share class unit. Initial unit price is used for the first NAV calculation. The price you enter is copied to the Market data info tab in the Security window in FA Back.

                        Date of initial price

                        (Set when you create a share class). The date for which the initial unit price is valid.

                        Format

                        Unit price multipliers to use in calculations. For more information about multipliers, see the description of the ??? tab in the Security window. Choose one of the options:

                        • Show price as is – Show the share class unit price without any multipliers.

                        • In percentage – Use "100" in both Multiplier fields for the security.

                        • Add manually – Enter the multipliers in the fields below.

                        Decimals for share class NAV

                        The number of decimals used in the share class NAV. If the field is empty, NAV is stored with 10 decimals.

                        Fee structure

                        The Fee structure tab defines fixed costs and fees applied on the share class level. Click Add fee and enter the information about the share class fees:

                        Fee formula

                        Fee formula used to calculate the fee and generate fee transaction.

                        Fee transaction type

                        Transaction type linked to the chosen fee formula.

                        Amount or %

                        Fee percentage or amount in the fund currency, depending on the fee formula.

                        Decimals

                        The number of decimals used in calculating fees (0 to 10). If you choose "Default", the number of decimals is taken from account settings in FA Back (see Portfolio window).Portfolio window

                        Calculate fee per security

                        Run the formula multiple times, for each security in the fund investment portfolio. Used, for example, for kickbacks.

                        Include in income distribution

                        Include the fee in the income distribution calculation.

                        Formula sets its own account

                        Set an account for the fee formula. For example, this can be useful with accrued account interest transactions over multiple accounts.

                        Cash transactions

                        Account settings to handle deposits and withdrawals.

                        Account number

                        A fund portfolio account to record deposits and withdrawals to. If the field is empty, the account in the fund settings or the default account is used.

                        Tax reporting

                        Settings for the Finnish tax reporting: fund company business ID and fund business ID.

                        FX hedging

                        FX hedging settings:

                        FX hedge transaction type

                        Transaction type for accrued FX hedge transaction.

                        Portfolio

                        Portfolio that contains the FX forward.
